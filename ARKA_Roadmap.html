Why It Looks Like a Student Project Right Now
These are the UX/product signals that give it away
Problem 01
No Landing Page
The site opens directly to a login form. Real products have a hero page that explains what the product does before asking you to sign up. First impression is everything.
Problem 02
It's an Insurance App With No Claims
The entire purpose of ARKA is automated claims. There is literally no claims page. A judge will ask "so where do I see my claim?" and the answer is nowhere.
Problem 03
Flat Fees ≠ Our Own README
Our README says premiums are 1–3% of weekly income. The app shows ₹9/₹49/₹149. These are completely different models. Judges read the docs.
Problem 04
Risk Gauge Does Nothing
There's a beautiful SVG gauge showing a risk score. But it doesn't affect the premium, doesn't trigger any action, and isn't explained in context. It's a visual orphan.
Problem 05
No Admin Side = No Insurer POV
Every judge will ask: "Who manages fraud on the other side?" Right now the answer is no one. No fraud queue, no zone map, no analytics. There's only a worker app.
Problem 06
1000+ Inline Styles
The code uses inline style={{}} props everywhere instead of CSS classes. This is a maintainability red flag if judges look at the repo, and makes the UI inconsistent.
